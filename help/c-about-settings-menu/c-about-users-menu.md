# 사용자 메뉴 정보{#about-the-users-menu}

사용자 메뉴를 사용하여 사용자를 보고 추가하고, 역할을 보고 추가하고, 역할 구성원을 변경할 수 있습니다. 사용자 메뉴에서 작업을 수행하려면 관리자 권한이 있는 사이트 검색/머천다이징 사용자여야 합니다.

## 계정 사용자 보기 {#task_FDDF30EE23C548DF8CFBB2FB2605303C}

페이지를 사용하여 모든 기존 계정 사용자를 볼 수 [!DNL View Users] 있습니다. 계정 사용자를 제거할 수도 있습니다(계정 소유자 제외).

선택된 계정 사용자만 사용자를 제거하거나 사용자 역할을 수정할 수 **[!UICONTROL User Admin]** 있습니다.

계정 소유자를 제거하려면 먼저 계정 소유권을 다른 사용자에게 양도해야 합니다.

소유권을 이전하면 다른 사용자와 마찬가지로 계정을 제거할 수 있습니다. 제거된 사용자는 상태 변경을 알리는 전자 메일을 받습니다.

다른 [계정 사용자에게](../c-about-settings-menu/c-about-account-options-menu.md#task_47E3C66CC6374274830DA10171E0CF17)계정 소유권 이전을 참조하십시오.

>[!NOTE]
>
>이 작업을 수행하려면 관리자 권한이 있는 사이트 검색/머천다이징 사용자여야 합니다.

**계정 사용자를 보려면**

1. 제품 메뉴에서 > **[!UICONTROL Settings]** > **[!UICONTROL Users]** > **[!UICONTROL View Users]**&#x200B;을 클릭합니다.
1. (선택 사항) [!DNL User Admin] 열 머리글에서 계정 사용자를 제거하거나 계정 사용자 역할을 편집할 수 있도록 하려는 계정 사용자를 선택합니다.
1. (선택 사항) [!DNL Remove?] 열 머리글에서 제거할 계정 사용자를 선택합니다.
1. 클릭 **[!UICONTROL Save Changes]**.
1. (선택 사항) [!DNL View Users] 페이지에서 하이퍼링크 역할 이름을 클릭합니다. 사용자를 역할에 할당하거나 역할에서 사용자 할당을 취소할 수 있는 [!DNL Role Membership] 페이지가 열립니다.

   역할 [구성원](../c-about-settings-menu/c-about-users-menu.md#task_DAC596AAFFCE4EF0BE68CEEF7E365414)구성을 참조하십시오.

## 계정 사용자 추가 {#task_176C463A0C0849B29245C28EC9876326}

이 [!DNL Add Users] 페이지를 사용하여 새 계정 사용자를 사이트 검색/머천다이징에 추가할 수 있습니다.

새 사용자의 이메일 주소만 필요합니다. 새 사용자가 계정에 추가되면 새 사용자에게 계정 정보가 전송됩니다.

기본적으로 새 사용자는 사용자 관리자로 지정되지 않습니다. 사용자 관리자는 역할을 정의 및 편집하고 다른 사용자를 제거할 수 있습니다. 새 사용자를 [!DNL Add Users] 페이지에서 사용자 관리자로 지정할 수 있습니다. 또는 [사용자 보기] 페이지를 사용하여 새 사용자를 사용자 관리자로 만들 수 있습니다.

계정 [사용자](../c-about-settings-menu/c-about-users-menu.md#task_FDDF30EE23C548DF8CFBB2FB2605303C)보기를 참조하십시오.

지정하는 전자 메일 주소에는 ASCII 문자만 포함되어야 합니다. 표준 알파벳을 사용합니다(a...z) 문자나 숫자(0.9)와 사용자 이름을 도메인에서 분리하는 데 사용되는 `@` 문자가 정확히 하나의 문자입니다. 등장 인물들, `_`, `+``-`, `.`, `!`, `#`, `$`, `'`, `%`,, `&``*``=``?``^``{``}` ,거세,전경,전경, 그리고 전경 또한 허용된다. 전자 메일 주소를 `.`

새 사용자가 이미 Adobe 고객이 아닌 경우 해당 사용자에 대한 고객 로그인을 만들 것인지 묻는 메시지가 표시됩니다. 새 사용자에게 로그인 암호 및 확인 메시지가 전송됩니다. 새 사용자가 처음 로그인하면 고객 프로필을 채웁니다.

선택적으로 [!DNL Add Users] 페이지에서 하이퍼링크 역할 이름을 클릭할 수 있습니다. 사용자를 역할에 할당하거나 역할에서 사용자 할당을 취소할 수 있는 [!DNL Role Membership] 페이지가 열립니다.

역할 [구성원](../c-about-settings-menu/c-about-users-menu.md#task_DAC596AAFFCE4EF0BE68CEEF7E365414)구성을 참조하십시오.

>[!NOTE]
>
>이 작업을 수행하려면 관리자 권한이 있는 사이트 검색/머천다이징 사용자여야 합니다.

**계정 사용자를 추가하려면**

1. 제품 메뉴에서 > **[!UICONTROL Settings]** > **[!UICONTROL Users]** > **[!UICONTROL Add Users]**&#x200B;을 클릭합니다.
1. 페이지의 [!DNL Add Users] 필드 및 [!DNL User's Email] [!DNL User's Email (again)] 필드에 새 사용자의 이메일 주소를 입력합니다.
1. (선택 **[!UICONTROL User Administrator]** 사항) 새 계정 사용자에게 계정 사용자를 제거하거나 계정 사용자 역할을 편집할 수 있도록 하려면 선택합니다.
1. 표에서 새 계정 사용자에게 할당할 역할을 [!DNL Roles for this User] 선택합니다.
1. 클릭 **[!UICONTROL Add User]**.

## 계정에 대해 존재하는 역할 보기 {#task_4EAE1D018F384691A083AD51E5CE58DC}

현재 로그인한 계정에 대해 현재 존재하는 모든 역할을 표시할 수 [!DNL View Roles] 있습니다.

계정에 대한 역할이 없는 경우 사용자 인터페이스에 이 문제가 표시됩니다. 역할 추가를 사용하여 역할을 만들고 추가할 수 있습니다.

계정에 [새 역할](../c-about-settings-menu/c-about-users-menu.md#task_E148D02275DE4F899BA79736A29895AB)추가를 참조하십시오.

역할을 제거해도 역할에 할당된 계정 사용자는 삭제되지 않습니다.

>[!NOTE]
>
>이 작업을 수행하려면 관리자 권한이 있는 사이트 검색/머천다이징 사용자여야 합니다.

**계정에 대해 존재하는 역할을 보려면**

1. 제품 메뉴에서 > **[!UICONTROL Settings]** > **[!UICONTROL Users]** > **[!UICONTROL View Roles]**&#x200B;을 클릭합니다.
1. (선택 사항) 테이블의 [!DNL Remove?] 열 머리글에서 제거할 역할을 선택합니다.
1. 클릭 **[!UICONTROL Save Changes]**.
1. (선택 사항) 다음 중 하나를 수행합니다.

   * 테이블의 [!DNL Role Name] 열 머리글에서 하이퍼링크된 역할 이름을 클릭합니다. 사용자를 역할에 할당하거나 역할에서 사용자 할당을 취소할 수 있는 [!DNL Role Membership] 페이지가 열립니다.

      역할 [구성원](../c-about-settings-menu/c-about-users-menu.md#task_DAC596AAFFCE4EF0BE68CEEF7E365414)구성을 참조하십시오.

   * 표의 [!DNL Edit] 열 머리글에서 연필 아이콘을 클릭합니다. 역할 이름 변경, 설명 변경 등이 가능한 [!DNL Edit Role] 페이지가 열립니다.

      역할 [편집을 참조하십시오](../c-about-settings-menu/c-about-users-menu.md#task_13875C2464034CE387285640412E1B59).

## 역할 편집 {#task_13875C2464034CE387285640412E1B59}

기존 역할의 이름을 변경하고, 해당 설명을 변경하고, 역할 액세스를 부여할 사용자 인터페이스의 영역만 선택할 수 있습니다.

계정에 [새 역할](../c-about-settings-menu/c-about-users-menu.md#task_E148D02275DE4F899BA79736A29895AB)추가를 참조하십시오.

>[!NOTE]
>
>이 작업을 수행하려면 관리자 권한이 있는 사이트 검색/머천다이징 사용자여야 합니다.

**역할을 편집하려면**

1. 제품 메뉴에서 > **[!UICONTROL Settings]** > **[!UICONTROL Users]** > **[!UICONTROL View Roles]**&#x200B;을 클릭합니다.
1. 표의 [!DNL Edit] 열 머리글에서 변경할 역할 옆에 있는 연필 아이콘을 클릭합니다.
1. (선택 사항) 다음 중 하나를 수행합니다.

   * 원하는 경우 [!DNL Role Name] 텍스트 필드에 새 이름을 입력합니다. *는 역할에 이 필드가 필수임을 나타냅니다.
   * 원할 경우 [!DNL Description] 텍스트 필드에 역할에 대한 새 설명을 입력합니다.
   * 그룹 상자에서 역할에 액세스하거나 차단할 기능을 선택하거나 선택 취소합니다.

1. 클릭 **[!UICONTROL Save Changes]**.

## 계정에 새 역할 추가 {#task_E148D02275DE4F899BA79736A29895AB}

이 [!DNL Add Roles] 페이지를 사용하여 계정 사용자에게 권한을 보다 쉽고 빠르게 할당할 수 있습니다.

예를 들어 홍보 부서의 각 구성원에게 사이트 검색/머천다이징 작업에 대한 액세스 권한을 개별적으로 부여할 수 있습니다. 그러나 이러한 사용자를 &quot;PR&quot; 역할에 추가한 다음 전체 역할에 작업을 할당하는 것이 훨씬 더 효율적입니다.

각 역할 이름은 고유해야 합니다. 대시, `"-"`밑줄, 마침표 등 영숫자 문자와 일반 기호를 사용할 수 `"_"``"."` 있습니다. 이름은 밑줄 또는 마침표로 시작할 수 없습니다.

테이블의 [!DNL Users in This Role] 열 헤더 아래에서 선택적으로 사용자의 하이퍼링크된 이메일 주소를 클릭할 수 있습니다. 사용자를 역할에 할당하거나 역할에서 사용자 할당을 취소할 수 있는 특정 사용자에 대해 [!DNL Role Membership] 페이지가 열립니다.

역할 [구성원](../c-about-settings-menu/c-about-users-menu.md#task_DAC596AAFFCE4EF0BE68CEEF7E365414)구성을 참조하십시오.

테이블의 [!DNL Roles] 열 헤더 아래에서 하이퍼링크된 역할 이름을 선택적으로 클릭할 수 있습니다. 선택한 역할에 사용자를 할당하거나 선택한 역할에서 사용자 할당을 취소할 수 있는 [!DNL Role Membership] 페이지가 열립니다.

>[!NOTE]
>
>이 작업을 수행하려면 관리자 권한이 있는 사이트 검색/머천다이징 사용자여야 합니다.

**계정에 새 역할을 추가하려면**

1. 제품 메뉴에서 > **[!UICONTROL Settings]** > **[!UICONTROL Users]** > **[!UICONTROL Add Roles]**&#x200B;을 클릭합니다.
1. 페이지의 [!DNL Add Roles] 필드에 [!DNL Role Name] 역할의 이름을 입력합니다.
1. (선택 사항) [!DNL Description] 필드에 역할을 적절히 설명하는 문장을 입력합니다.
1. 각 이메일 주소 왼쪽에 있는 상자를 선택하여 역할에 속한 사용자를 선택합니다.
1. 클릭 **[!UICONTROL Add Role]**.

## 역할 구성원 구성 {#task_DAC596AAFFCE4EF0BE68CEEF7E365414}

사용자별로 역할을 볼 때 개별 사용자 그룹 멤버십을 관리할 수도 있습니다.

계정에 [새 역할](../c-about-settings-menu/c-about-users-menu.md#task_E148D02275DE4F899BA79736A29895AB)추가를 참조하십시오.

계정에 [대해](../c-about-settings-menu/c-about-users-menu.md#task_4EAE1D018F384691A083AD51E5CE58DC)존재하는 역할 보기를 참조하십시오.
